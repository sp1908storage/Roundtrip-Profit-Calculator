# Freight Cost Bot Prototype

Прототип Python-бота для сбора данных по прямым и обратным рейсам, проверки заявки,
расчета себестоимости и подготовки ответа менеджеру.

## Что уже есть

- Нормальная структура данных для круго-рейса и отдельных рейсов.
- Пошаговый диалог: бот спрашивает только недостающие поля.
- Проверки обязательных полей, веса, маршрута, страны и НДС.
- Расчет себестоимости через настраиваемый конфиг.
- Заготовка AI-парсера для извлечения данных из свободного текста.
- Заготовка интеграции с Google Sheets.
- CLI-режим для тестирования без Telegram/WhatsApp.

## Быстрый запуск

```powershell
python main.py
```

Можно вставить исходное описание перевозки одной строкой. Если часть данных не указана,
программа задаст уточняющие вопросы.

## Запуск Telegram-бота

Создайте локальный `.env` по примеру `.env.example`, заполните `TELEGRAM_BOT_TOKEN`,
Google Sheets-поля и при необходимости `OPENAI_API_KEY`.

```powershell
python run_telegram.py
```

Бот работает в polling-режиме, поэтому отдельный сервер и webhook не нужны.

## Важные настройки

Формулы и базовые ставки лежат в `cost_bot/config.py`. Сейчас там стоят демонстрационные
значения, чтобы прототип можно было запускать сразу. Для боевого расчета нужно заменить:

- расход топлива;
- цену топлива;
- стоимость обслуживания на км;
- зарплату/суточные/постоянные расходы на км;
- платные дороги;
- коэффициенты для международных рейсов;
- правила запрещенных направлений.

## Где подключать AI

AI должен возвращать нормализованный JSON по схеме `cost_bot/models.py`.
Точка подключения: `cost_bot/ai_parser.py`. Сейчас адаптер использует OpenAI Responses API
со строгой JSON-схемой. Если `OPENAI_API_KEY` не задан, прототип работает в ручном режиме
и задает уточняющие вопросы.

## Где подключать Google Sheets

Точка подключения: `cost_bot/sheets.py`.

## Что нужно уточнить для боевой версии

1. Платформа бота: Telegram, WhatsApp, Bitrix24, веб-чат или другое.
2. Точные формулы себестоимости и статьи расходов.
3. Источник пробегов и сухопутной проверки: готовые километры от менеджера или API карт.
4. Правила запрещенных направлений и международных коэффициентов.
5. Структура Google Sheets: один лист, несколько листов, поля, права доступа.

## Предлагаемая боевая архитектура

1. Канал бота принимает сообщение менеджера.
2. `ai_parser` извлекает возможные поля из свободного текста.
3. `dialogue` задает только недостающие вопросы.
4. `validators` выполняет строгие проверки без AI.
5. `calculator` считает расходы по утвержденным формулам.
6. `sheets` записывает исходную заявку, нормализованные поля, расчет и комментарий.
7. Бот возвращает менеджеру себестоимость, прибыль и предупреждения.
